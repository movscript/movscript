# Movscript Product Experience Optimization Plan

本文档描述 Movscript 当前产品体验的主要问题、目标体验和可落地的优化方案。重点不是新增更多功能，而是让已有能力形成清晰、可完成、可复用的生产路径。

## 当前体验诊断

Movscript 已经具备大量能力，但普通用户第一次进入时会遇到几个问题：

1. **入口过多，主线不够突出**
   - 侧边栏同时展示项目、内容实体、工作台、工具、文件、管理、未使用入口。
   - 用户能看到很多能力，但不容易判断第一步应该做什么。

2. **页面按数据实体组织，而不是按生产阶段组织**
   - `segments`、`scene-moments`、`contents`、`asset-slots` 等对开发者和内部数据模型清晰，但对创作者来说偏抽象。
   - 用户更自然的理解方式是：剧本、拆解、分镜、素材、生成、成片。

3. **项目首页已经有看板雏形，但还没有成为默认工作指挥台**
   - `ProjectHomePage` 已经聚合了项目数据、进度和工作入口。
   - 但侧边栏仍把所有子页面平铺出来，削弱了项目首页的中心地位。

4. **生产预览页能力强，但流程负担较重**
   - `ProductionPreviewPage` 覆盖了剧本草稿、AI 分析、分镜建议、关键帧候选、素材需求缺口、Agent planning。
   - 对新用户来说，页面更像功能集合，而不是清晰的分步骤向导。

5. **AI 配置与主流程的关系不够产品化**
   - 管理后台有模型配置、feature routing、debug。
   - 但普通用户在主流程里遇到“模型未配置”时，应该看到清晰的配置缺口和下一步按钮。

6. **调试能力过早暴露**
   - Agent Debug、Admin Debug、UI Preview、Unused routes 都在导航中可见。
   - 这些入口对开发者有价值，但会干扰普通生产用户。

## 目标体验

Movscript 的理想体验应围绕一个问题展开：

> 当前项目进行到哪一步？下一步最应该做什么？

用户进入产品后，应该形成这样的路径：

```text
选择或创建项目
  -> 项目首页看到当前生产阶段和下一步
  -> 导入/编辑剧本
  -> AI 拆解生产结构
  -> 审阅分镜和关键帧候选
  -> 处理素材需求缺口
  -> 确认生产预览
  -> 进入生成/画布/成片阶段
```

## 信息架构优化

### 当前问题

当前侧边栏更像完整功能目录，而不是产品导航。它暴露了太多中间实体和实验入口。

### 建议结构

默认侧边栏只保留 5 个主区：

1. **项目**
   - 项目首页
   - 剧本
   - 生产预览
   - 生成工作台
   - 交付

2. **资源**
   - 素材库
   - 任务队列

3. **画布**
   - 工作流画布
   - 工作流模板

4. **团队**
   - 成员
   - 审批
   - 用量

5. **设置**
   - 模型与 AI
   - 插件
   - 系统设置

开发/调试入口放到：

- 设置中的“开发者模式”
- 或用户偏好里的 `Show developer tools`
- 或仅在 `super_admin` 且 debug flag 开启时显示

### 实体页面处理

以下页面不建议默认全部放在主导航：

- `segments`
- `scene-moments`
- `creative-references`
- `asset-slots`
- `reference-relations`
- `contents`
- `final-videos`

它们应该作为项目首页或生产预览中的“详情入口”，而不是普通用户必须理解的一级菜单。

## 项目首页优化

项目首页应该成为每个项目的默认落点。

### 应显示的信息

- 当前项目阶段：策划、剧本拆解、素材准备、生成中、剪辑/交付、完成。
- 下一步建议：一个主按钮，不超过两个次要按钮。
- 生产包完整度：
  - 剧本是否存在
  - 是否已拆解
  - 分镜是否确认
  - 关键帧是否确认
  - 素材需求缺口是否处理
  - 是否可以进入生成
- 待处理事项：
  - 待接受的 AI 分镜建议
  - 待确认关键帧
  - 未解决素材需求缺口
  - 失败任务
- 最近活动：
  - AI 任务
  - 人工确认
  - 资源上传
  - 成员操作

### 下一步按钮规则

项目首页应该根据数据自动给出最强操作：

| 状态 | 主按钮 |
| --- | --- |
| 没有剧本 | 添加剧本 |
| 有剧本但无草稿 | 开始生产预览 |
| 有草稿未分析 | 解析剧本 |
| 有 AI 候选未审阅 | 审阅 AI 建议 |
| 有素材需求缺口 | 处理素材需求缺口 |
| 预览已确认 | 进入内容生成 |
| 有失败任务 | 查看失败任务 |

这样可以让用户不用理解全部页面结构，也能持续推进。

## 生产预览页优化

`ProductionPreviewPage` 应从“大型综合页”改造成阶段化体验。

### 推荐阶段

1. **剧本来源**
   - 选择剧本版本
   - 粘贴/编辑剧本
   - 保存草稿

2. **结构理解**
   - 运行 AI 分析
   - 展示片段、情节、角色/地点/道具信号
   - 允许人工修正

3. **分镜决策**
   - 展示 AI 分镜建议
   - 批量接受、拒绝、修改
   - 显示已采纳数量

4. **预演候选**
   - 生成关键帧候选
   - 生成预演时间线
   - 展示素材需求缺口

5. **确认生产包**
   - 检查剩余阻塞项
   - 确认后写入正式项目结构
   - 引导进入生成工作台

### 交互原则

- 每个阶段只突出一个主操作。
- 未满足前置条件时，显示原因和修复按钮。
- 复杂候选列表默认折叠摘要，展开后再做细节编辑。
- 接受/拒绝 AI 结果时，显示结果会写入哪里。
- 确认生产包前，给用户一个检查清单。

## 首次使用体验

### 首次打开

如果用户没有项目：

- 显示“创建第一个项目”。
- 创建后直接进入项目首页，不停留在项目列表。
- 项目首页显示“添加剧本”作为主按钮。

### 首次 AI 使用

如果用户没有可用模型：

- 不要只显示失败 toast。
- 在当前页面显示配置缺口：
  - 未配置 AI provider。
  - 未配置 text 模型。
  - 当前 feature 没有默认模型。
- 提供按钮：
  - 去配置模型
  - 使用 dry-run 模式体验流程

### 首次团队使用

团队模式下，管理员首次进入应看到：

- backend 状态
- 存储状态
- 模型状态
- 用户/组织状态
- 是否已配置 gateway key

普通成员不应看到 provider key 配置细节。

## 导航与模式

Movscript 至少应区分三种用户模式：

1. **创作者模式**
   - 默认模式。
   - 只显示项目、剧本、生产预览、资源、生成、交付。

2. **制片/管理员模式**
   - 显示团队、用量、模型、权限、审计。

3. **开发者模式**
   - 显示 Agent Debug、Admin Debug、UI Preview、raw JSON、prompt、tool call。

不要让所有模式的入口同时出现在一个侧边栏里。

## AI 体验优化

### AI 结果标识

所有 AI 生成内容都应明确标识：

- AI 生成
- 人工修改
- 已确认
- 已拒绝
- 来源模型
- 生成时间

### AI 操作反馈

每个 AI 操作都要有统一状态：

- 准备中
- 排队中
- 生成中
- 等待审阅
- 已确认
- 失败
- 可重试

### AI 失败处理

失败提示应包含：

- 用户能理解的原因。
- 是否可重试。
- 是否因为模型未配置、额度不足、资源不可访问或 provider 错误。
- 一个明确的修复入口。

### AI 成本提示

团队模式下，关键生成操作前可显示：

- 预计模型
- 预计成本范围
- 将写入的目标
- 是否需要审批

## 资源体验优化

资源库不应只是文件列表，而应该服务生产。

推荐组织方式：

- 项目资源
- 角色
- 场景
- 道具
- 参考图
- 关键帧
- 视频片段
- 未归档

资源卡片应显示：

- 使用位置
- 来源任务
- 是否被确认
- 是否有派生资源
- 可用于哪些生成工具

素材需求缺口应能直接转化为：

- 上传资源
- 从资源库选择
- 调用 AI 生成
- 标记为暂不处理

## Agent 体验优化

Agent 不应该首先以 Debug Page 出现。普通用户应该看到的是“项目助手”。

项目助手的任务：

- 解释当前项目状态。
- 告诉用户下一步。
- 查找项目内容。
- 创建草稿。
- 总结 AI 候选。
- 生成 production plan。

Agent Debug 保留给开发者，但普通用户界面只显示：

- 正在读取项目
- 已生成计划
- 已创建草稿
- 需要你确认
- 已完成

## 插件体验优化

插件应从“工具列表”变成“能力扩展”。

推荐展示：

- 插件名称
- 插件提供的工具
- 需要的权限
- 是否可用于当前项目阶段
- 最近运行结果

插件安装时需要明确：

- 会读取哪些数据
- 会写入哪些数据
- 是否会调用 AI
- 是否会访问资源文件

## 可立即实施的 Quick Wins

1. **创建项目后直接进入项目首页**
   - 现在创建成功后只刷新项目列表。
   - 建议创建成功后设置当前项目并跳转 `/project-home`。

2. **侧边栏默认隐藏 Unused 和 Debug 入口**
   - `Unused routes kept visible for audit` 不应对普通用户可见。
   - Agent Debug、Admin Debug、UI Preview 应进入开发者模式。

3. **把项目首页设为强入口**
   - 用户切换项目后进入 `/project-home`。
   - 项目首页主按钮根据状态跳转下一步。

4. **生产预览页加顶部阶段条**
   - 剧本来源、结构理解、分镜决策、预演候选、确认生产包。
   - 每个阶段显示完成/当前/阻塞。

5. **模型未配置时提供内联修复**
   - 在生产预览和生成工具页显示“去配置模型”按钮。
   - 支持 dry-run 模式用于体验流程。

6. **统一任务失败卡片**
   - 显示失败原因、可重试按钮、查看 debug 按钮。
   - 普通用户默认隐藏 provider raw response。

7. **把实体页面从主导航降级**
   - 内容实体保留为项目首页和生产预览的详情入口。
   - 侧边栏主导航只保留生产阶段。

## 中期体验改造

1. **项目阶段引擎**
   - 后端或前端统一计算项目当前阶段、阻塞项和推荐下一步。
   - 项目列表、项目首页、Header、Agent 都使用同一套结果。

2. **生产预览向导**
   - 将 ProductionPreviewPage 拆成阶段组件。
   - 每个阶段有清晰输入、输出、主操作和完成条件。

3. **配置健康中心**
   - 显示 backend、storage、model、runtime、gateway 的状态。
   - 给出修复动作。

4. **团队工作台**
   - 待我审阅。
   - 我发起的任务。
   - 失败任务。
   - 本周用量。

5. **统一审阅中心**
   - AI 分镜建议、关键帧候选、素材需求缺口、Agent 草稿都进入同一种审阅体验。

## 成熟体验标准

Movscript 的产品体验成熟，应达到：

- 新用户不读架构文档也能完成第一个生产预览。
- 普通用户不需要理解 MCP、adapter、model gateway、manifest。
- 每个页面都有明确主操作。
- 每个 AI 结果都有来源、状态和审阅动作。
- 每个失败都有下一步。
- 团队管理员可以配置模型和权限，成员只使用能力。
- Debug 信息完整存在，但不会干扰普通用户。

## 推荐实施顺序

1. 导航收束。
2. 项目首页下一步引擎。
3. 生产预览阶段化。
4. AI 配置缺口内联修复。
5. 任务失败体验统一。
6. 资源与素材需求缺口闭环。
7. Agent 从 Debug 转为项目助手。
8. 团队模式和权限体验。
